# Final Reflection

## 1. How does Git help in maintaining research traceability?

Git maintains a complete history of all changes made to a project. Each commit records what was changed, when it was changed, and who made the change. This allows researchers to trace the evolution of their work, revisit previous versions, and understand how results were derived. It ensures transparency and accountability in the research process.



## 2. Why is GitHub useful for academic collaboration?

GitHub provides a centralized platform where multiple contributors can work on the same project. It allows sharing of code, documentation, and datasets, while also supporting version control. Features like issues, pull requests, and discussions make collaboration structured and efficient, especially for research teams.



## 3. Why should researchers create branches instead of working directly on main?

Branches allow researchers to work on new features, experiments, or documentation without affecting the stable version of the project. This ensures that the main branch remains clean and reliable. It also enables safe experimentation and easy rollback in case of errors.



## 4. What is the role of Pull Requests in collaborative work?

Pull Requests (PRs) allow changes made in a branch to be reviewed before merging into the main branch. They facilitate discussion, feedback, and validation of work. PRs help maintain code quality and ensure that all changes are properly documented and reviewed.



## 5. Why are reproducibility files important in research repositories?

Reproducibility files, such as requirements.txt and methodology documents, ensure that other researchers can replicate the results of a project. They clearly define dependencies, execution steps, and assumptions, which are essential for validating and extending research work.


## 6. What is the purpose of creating a version tag such as v1.0.0?

A version tag marks a specific stable state of the project. It allows researchers to refer to a particular version of the work, ensuring consistency in results. Version tags are useful for sharing, publishing, and referencing research outputs.


## 7. Why should a repository include a citation file?

A citation file provides a standard way for others to reference the repository in their work. It ensures that the original author receives proper credit and encourages ethical reuse of research materials.


## 8. How does this semester’s Git/GitHub workflow support better PhD research practice?

This workflow promotes structured development, proper documentation, and reproducibility. It encourages maintaining clean project history, collaborating effectively, and presenting research in a professional manner. These practices are essential for high-quality PhD research, where transparency, traceability, and reproducibility are critical.
